# Pieza 15 - Codex en repos

## Objetivo

Usar Codex como herramienta de ingenieria sobre repos reales.

## Concepto

Codex sirve para:

- inspeccionar un repo
- modificar archivos
- correr comandos
- explicar errores
- proponer tests

Pero el pedido tiene que ser verificable.

## Formula

```text
objetivo + alcance + archivos relevantes + verificacion
```

## Ejemplo

```text
Agrega GET /health.
Lee primero la estructura.
No refactorees.
Verifica con curl.
```

## Tu problema

En `2-tu-trabajo.md`, escribi 3 pedidos para Codex:

- entender un repo
- agregar un endpoint chico
- revisar si tu solucion necesita tests

## Transferencia al demo

Codex entra como companero de ingenieria sobre tu repo, no como generador de
magia.

Un buen pedido para este proyecto incluye:

- objetivo concreto
- archivos relevantes
- limite de alcance
- comando de verificacion
- criterio de exito
